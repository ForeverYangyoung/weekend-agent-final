## 0. TL;DR (一句话定义)

> 构建一个 **「本地短时活动规划 + 执行」Agent**：用户用一句自然语言描述目标（人群、时间、约束），Agent 在几分钟内输出可执行方案，并**自动完成下单、预订、送礼、通知**等关键动作，形成端到端闭环。

**关键定性 (划重点)：**
- ✅ 这是 **Task-Completion Agent**，不是 Recommender / Chatbot
- ✅ 必须打通 **"想 → 查 → 定 → 通知"** 的全链路
- ❌ 不是搜索推荐 (评委原话："这不是搜索推")
- ❌ 不是单轮问答

## 1. 题目原文要素抽取

### 1.1 任务背景 (原文要素)

| 原文片段 | 工程含义 |
|---|---|
| "周六上午 9 点" | 触发时刻 (now) |
| "今天下午是空的" | 时间窗口约束: 下午, 4-6h |

| "别离家太远" | 地理范围约束 (默认 ≤ 同城/3-10km) |
| "帮我安排一下" | 触发动作: 启动 Planner |

### 1.2 两个测试场景 (必须覆盖)

#### 场景 A · 家庭场景
```yaml
participants:
  - role: 自己, age: 35
  - role: 老婆, age: 32, constraints: [减肥, 低卡]
  - role: 孩子, age: 5, constraints: [适合儿童, 安全]
size: 3
preferences_implicit:
  - 餐厅需有儿童座椅/儿童餐
  - 活动需 family-friendly
  - 老婆: 避开高热量、火锅、油炸
```

#### 场景 B · 朋友场景
```yaml
participants: 4 (男2 女2)
size: 4
preferences_implicit:
  - 适合年轻人聚会
  - 桌位需 ≥ 4 人
  - 可能需要"展览/citywalk/小吃街"等社交属性活动
```

### 1.3 三个动作触点 (原文要求 Agent 几分钟内完成)

```
触点 1: 规划 (Plan)
  └─ 输出: 一条"下午 4-6 小时综合方案"
     └─ 子任务: 去哪玩 → 玩完去哪吃 → 有没有额外活动

触点 2: 校验 (Verify)
  └─ 查餐厅是否符合人群需求
  └─ 查餐厅是否有空位
  └─ 查是否需要排队

触点 3: 增项 (Enrich)
  └─ 吃饭前后的活动：亲子乐园 / 展览 / citywalk / 小吃街

触点 4: 执行 (Execute) ← 最关键
  └─ 用户确认方案后:
     ├─ 一键预订餐厅
     ├─ 一键购票
     ├─ 送蛋糕/鲜花到指定餐厅
     └─ 把计划分享给小张 ("搞定了，下午X点出发，先去...")
```

| 交付物 | 必须 | 评估关注点 |
|---|---|---|
| Demo (命令行 / Web UI 均可) | ✅ | 能跑通 + 演示效果 |

| 设计文档 ≤ 2 页 | ✅ | Planning 策略 + 工具链路 + 异常处理 |

## 2. 工程语言重述 (System Definition)

### 2.1 系统形式化定义

```
INPUT:
  utterance:   string         # 自然语言目标
  context:     {
    now:       datetime,
    location:  GPS / 城市,
    user_id:   string,
    history:   optional[过往偏好]
  }

OUTPUT:
  plan:        Plan           # 结构化方案
  executions:  Execution[]    # 已落地动作的回执
  notifications: Notify[]     # 已发出的通知

WHERE:
  Plan = {
    summary:    string,
    timeline:   Step[],        # 时间轴上的步骤
    total_cost: number,
    total_time: duration,
    alternatives: Plan[]       # 备选方案 (Top-2)
  }

  Step = {
    type:       enum[玩, 吃, 路途, 活动, 送礼],
    start_at:   datetime,
    end_at:     datetime,
    poi:        POI,
    actions:    Action[]       # 需要执行的动作
  }

  Action = {
    tool:       string,
    payload:    object,
    status:     enum[pending, success, failed, rolledback],
    receipt:    object         # 执行回执 (订单号等)
  }
```

### 2.2 核心问题分解 (4 层)

```
Layer 1  意图理解 (NLU)
         自然语言 ──► 结构化 Query (人群/时间/预算/约束)

Layer 2  方案规划 (Planning)
         结构化 Query ──► 阶段骨架 [玩 → 吃 → 活动]
                       ──► 各阶段候选 (并发查询工具)
                       ──► 约束求解 + 组合 Top-K 方案

Layer 3  执行编排 (Orchestration)
         已确认方案 ──► 并发/串行执行多个工具调用
                    ──► 失败重试 + 回滚 + 替代方案

Layer 4  通知协同 (Notification)
         执行结果 ──► 生成可分享卡片/链接
                  ──► 发送给指定联系人
```

## 5. 用户故事 (User Story) - 必须覆盖的 Demo Case

### Story 1 · 家庭温情场景 (Happy Path)
> 作为一个周六上午的小明，我希望发一句"今天下午想带老婆孩子出去玩 4 小时"，Agent 给我一份方案：14:00-16:00 朝阳公园儿童乐园 → 16:30-18:00 XX低卡轻食 (已订位)，并把方案发给老婆微信。

**验收**：能跑通 search → reserve → share 三步闭环。

### Story 2 · 朋友聚会场景 (复杂约束)
> 作为小明，我说"下午约 4 个朋友 2 男 2 女出去玩"，Agent 输出：14:00 三里屯 citywalk → 17:00 XX 小酒馆 (已预订 4 人桌) → 19:00 看展。

**验收**：体现"4 人桌"约束被使用、地理顺路、增项活动。

### Story 3 · 异常自愈场景 (撒手锏 ⭐)
> 用户确认方案后，Agent 执行预订时遇到"该餐厅已满座"，自动触发替代方案搜索，找到 500m 内同档次餐厅并完成预订。

**验收**：能演示失败 → 自愈 → 成功的完整链路。这是拉分项。

### Story 4 · 送礼增项 (业务深度)
> "今天老婆生日"——Agent 在方案里追加 "send_gift_to_restaurant(蛋糕, 18:00 送达 XX餐厅)"，并展示送达回执。

**验收**：体现 Agent 不止"安排"，还能"代办"。

## 6. Tool 接口契约 (Mock API 规范)

> 硬件同学按这个清单实现 Mock Server。所有接口走 HTTP JSON，端口 `:8001`。

### 6.1 信息查询类 (Read)

```yaml
GET /api/poi/search
  query:
    category:   string  # 餐厅|景点|亲子|展览|citywalk
    location:   string
    filters:    object  # {人均: 100-200, 标签: [低卡, 儿童友好]}
    size:       int
  response:
    pois:       POI[]

GET /api/poi/{id}
  response: POI详情 (营业时间/人均/标签/图片)

GET /api/restaurant/{id}/table
  query: { time: datetime, people: int }
  response: { available: bool, waiting_minutes: int }

GET /api/restaurant/{id}/queue
  response: { current_waiting: int, estimated_minutes: int }

GET /api/weather
  query: { location, time }
  response: { condition, temp, suitable_outdoor: bool }

GET /api/route
  query: { from, to, mode: 步行|地铁|打车 }
  response: { duration_min, distance_km, path }
```

### 6.2 执行类 (Write) ★ 核心

```yaml
POST /api/restaurant/reserve
  body: { poi_id, time, people, contact, idempotency_key }
  response: { order_id, status, qr_code }

POST /api/ticket/book
  body: { activity_id, count, idempotency_key }
  response: { order_id, status, ticket_codes[] }

POST /api/food/order
  body: { poi_id, items[], delivery_address?, idempotency_key }
  response: { order_id, eta_minutes, total }

POST /api/gift/send
  body: { gift_type: 蛋糕|鲜花, deliver_to_poi_id, deliver_time, message }
  response: { order_id, courier_phone }

POST /api/notify/share
  body: { plan_id, recipient: string, channel: 微信|短信 }
  response: { sent: bool, preview_url }
```

### 6.3 回滚类 (Compensation)

```yaml
POST /api/order/{id}/cancel
  response: { cancelled: bool, refund_amount }
```

### 6.4 故障注入规约 (评测关键!)

Mock Server **必须**支持以下故障：
| 接口 | 故障概率 | 故障形式 |
|---|---|---|
| `restaurant/{id}/table` | 15% | `available: false` |
| `restaurant/reserve` | 10% | 返回 409 满座 |
| `ticket/book` | 5% | 返回 410 售罄 |
| 任意接口 | 3% | 超时 (5s) |

> 通过 query param `?force_fail=table_full` 可指定故障，方便 Demo 演示。

## 7. 异常场景枚举 (Error Handling Spec)

| 异常 | 触发条件 | Agent 应对策略 |
|---|---|---|
| E-01 餐厅满座 | reserve 返回 409 | 在 1km 内同类目同档次搜索替代，自动重试 ≤ 2 次 |
| E-02 活动售罄 | book 返回 410 | 切换到备选方案 (Plan B) 中的同时段活动 |
| E-03 接口超时 | timeout > 5s | 指数退避重试 (1s, 2s, 4s)，3 次失败后 fallback |
| E-04 用户改口 | 二次输入修正条件 | 增量更新 Query，不重新规划全程 |
| E-05 隐含冲突 | "减肥 + 火锅店" | LLM 二次确认，提示矛盾并给建议 |
| E-06 时间不够 | 玩 + 吃 + 路途 > 时间窗 | 砍掉优先级最低的步骤 (通常是增项活动) |
| E-07 工具调用失败 | LLM 输出不合法工具名/参数 | 用 Pydantic 校验并 reprompt |

## 8. 数据模型 (Domain Model)

```python
class POI:
    id: str
    name: str
    category: Literal["餐厅","景点","亲子","展览","citywalk","酒吧"]
    location: GeoPoint
    avg_price: int
    tags: List[str]               # ["低卡","儿童友好","适合聚会"]
    open_hours: TimeRange
    rating: float

class Constraint:
    size: int                     # 人数
    age_range: Tuple[int,int]     # 年龄区间
    diet: List[str]               # ["低卡","素食"]
    budget: int                   # 预算/人
    time_window: TimeRange
    location_radius_km: float
    preferences: List[str]

class Step:
    type: Literal["玩","吃","路途","活动","送礼"]
    start_at: datetime
    end_at: datetime
    poi: Optional[POI]
    actions: List[Action]

class Action:
    tool: str
    payload: dict
    status: Literal["pending","success","failed","rolledback"]
    receipt: Optional[dict]
    error: Optional[str]

class Plan:
    plan_id: str
    summary: str
    timeline: List[Step]
    total_cost: int
    constraints: Constraint
    alternatives: List["Plan"]
```

| **Planner 硬约束过滤** | `agents/planner._passes_hard_filter`：亲子玩 / 低卡吃 / 距离；砍光则 soft fallback。 | `03-Planner+Scorer设计文档.md` Step 2 · `planner/filters.py` |
| **规则选店（可解释）** | Researcher 按 `breakdown.total` 排序；Planner 取过滤后首位；`Plan.score` = 各 stage primary 均分。 | `scoring/rules.py` + `Researcher Agent 设计思路.md` |
| **两种阶段顺序** | `_ORDER_VARIANTS`：`玩→吃` / `吃→玩`；`Plan.order_label`；加餐锚在「吃」+90min。 | `03-Planner+Scorer设计文档.md` Step 0（**规则 2 种**，非 LLM） |
| **Top-2 方案** | `build_plans(top_k=2)` → `plan` + `plan_alternatives`；notifier 行程卡「备选方案」。 | `03-Planner+Scorer设计文档.md` |
| **Tool 全链路 HTTP** | `registry.invoke` → `http_client` → `mock_meituan/routes`；DryRun / Executor / Compensator 同源。 | `planner/tool_hub.py` + `01-题目工程拆解.md` Mock API |
| **失败注入 + 补偿重规划** | `force_failure` → 写请求 `force_fail` → 409；`cancel_order` HTTP；Planner `blocked` 换备选 POI。 | `02.架构和agent.md` §7.3 · `nodes/compensator.py` |
| **Critic 规则校验** | `nodes/critic.py`：低卡 / 亲子 / 阶段完整；不通过回 Planner（未超迭代上限）。 | `nodes/critic.py` |
| **统一 trace** | `roles.trace_line`；四角色名与架构文档一致。 | `02.架构和agent.md` §7 · `planner/trace.py` |
| **Demo / 答辩开关** | `demo --scene` / `--fail 吃` / `--history`；打印 Mock 内联 vs TCP。 | 工程约定 · `docs/mock-api.md` |
| **4 人订座陷阱 + 预检重规划** | `poi_rest_201` + `people>=4` 返满座；`dry_run` 失败 → Planner 拉黑换备选 | `mock_meituan/backend.py` · `graph.py` |
| **玩/吃顺路距离** | `abs(dist_play-dist_eat) <= 3km` 才成方案 | `agents/planner.py` |
| **Critic 低卡/菜系硬红线** | `severity=block` 强制重规划 | `nodes/critic.py` |
| **加餐送至餐厅闭环** | `deliver_to_poi_id` 写入 Tool payload 与订单回执 | `planner.py` · `plan_mapping.py` · `registry.py` |
